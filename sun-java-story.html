Sun, Java & Oracle: Vom Hardware-Giganten zum Software-Asset

Die Geschichte von Sun Microsystems ist eine Lehrstunde über den Wandel von Hardware- zu Software-zentrierten Geschäftsmodellen.

Akt 1: Der Aufstieg von Sun und Java (ca. 1982 - 2000)

Sun Microsystems: Der König der Workstation

Während IBM und die PC-Klone den Desktop-Markt eroberten, dominierte Sun Microsystems eine andere, sehr profitable Nische: den Markt für High-End-Workstations und Server. Ihre Maschinen, basierend auf der eigenen SPARC-Prozessorarchitektur und dem robusten Solaris-Betriebssystem (eine UNIX-Variante), waren die Arbeitspferde für Ingenieure, Wissenschaftler und vor allem für die Infrastruktur des frühen Internets. Sie waren teuer, leistungsstark und galten als Inbegriff professioneller, zuverlässiger Computertechnik.

Java: Der Geniestreich

In dieser fragmentierten Welt (Windows auf dem Desktop, verschiedene UNIX-Systeme auf Servern, Mac im Kreativbereich) entwickelte Sun Mitte der 90er Jahre die Programmiersprache Java. Die zentrale Idee war revolutionär: "Write Once, Run Anywhere". Ein in Java geschriebenes Programm wurde für eine "Java Virtual Machine" (JVM) kompiliert. Solange es für ein Betriebssystem eine JVM gab, lief das Java-Programm darauf. Java wurde zur dominanten Sprache für plattformunabhängige Unternehmensanwendungen und für das aufkommende World Wide Web.

Akt 2: Der Niedergang von Sun und die Kommerzialisierung durch Oracle (ca. 2001 - 2010)

Suns Geschäftsmodell, das auf dem Verkauf teurer, proprietärer Hardware basierte, wurde von zwei Seiten angegriffen:

  • Das Platzen der Dotcom-Blase (2001): Viele von Suns Kunden waren Internet-Startups, die über Nacht verschwanden.
  • Der Aufstieg von "Good Enough": Viel wichtiger war jedoch, dass die günstigen x86-Prozessoren (Intel, AMD) immer leistungsfähiger wurden. In Kombination mit dem kostenlosen, robusten Betriebssystem Linux konnten Unternehmen plötzlich Server-Systeme bauen, die "gut genug" waren, aber nur einen Bruchteil einer Sun-Workstation kosteten. Der Markt für teure Spezialhardware brach zusammen.

Die Übernahme durch Oracle (2010)

Oracle, ein Gigant im Datenbankgeschäft, erkannte die Situation perfekt. Sie wussten, dass Suns Hardware-Geschäft am Ende war, aber sie sahen auch den unschätzbaren Wert von Suns Software-Kronjuwel: Java. Fast alle großen Unternehmensanwendungen, die auf Oracle-Datenbanken zugriffen, waren in Java geschrieben. Mit der Übernahme von Sun kaufte sich Oracle die Kontrolle über die wichtigste Unternehmens-Programmiersprache der Welt. Dies sicherte ihr eigenes Kerngeschäft ab und gab ihnen die Möglichkeit, Java strategisch zu nutzen und zu monetarisieren.

Epilog: Der Rechtsstreit und der Aufstieg von Kotlin

Die Kommerzialisierung von Java durch Oracle hatte eine direkte und weitreichende Folge: den jahrelangen, milliardenschweren Rechtsstreit mit Google über die Nutzung von Java-APIs im Android-Betriebssystem. Obwohl Google den Fall letztendlich vor dem Obersten Gerichtshof der USA gewann, schuf der Rechtsstreit eine Dekade der Unsicherheit und ein enormes strategisches Risiko.

Als Reaktion darauf suchte Google nach einer Alternative, um seine Abhängigkeit von Oracle zu verringern. Die Lösung fand sich in Kotlin, einer modernen, von JetBrains entwickelten Sprache, die zu 100% mit Java interoperabel ist. Kotlin bot nicht nur technische Vorteile wie eine prägnantere Syntax und Nullsicherheit, sondern vor allem strategische Sicherheit. 2017 kündigte Google die offizielle Unterstützung für Kotlin auf Android an und erklärte es 2019 zur bevorzugten Sprache für die Android-Entwicklung. Der Aufstieg von Kotlin ist somit eine direkte Konsequenz aus dem Wandel von Java von einer offenen Plattform unter Sun zu einem kommerzialisierten und juristisch verteidigten Asset unter Oracle.
